# BitgetFollow 自动跟单交易系统知识库

这份文档记录了 BitgetFollow 系统的核心架构与关键防坑机制。系统于 **2026年02月** 进行了底层架构的全面重构，为了后续维护和迭代不埋坑，特此沉淀所有核心修复与运作机制。

## 1. 系统核心通讯架构 (高并发轮询)

系统通过轮询方式抓取交易员公网页面的持仓数据，而非通过 Bitget 官方的 websocket（不需要交易员给出他们的 API Key，也不需要和交易员建立“官方跟单关系”）。
- **并发请求**: 为了解决多交易员导致的轮询排队延迟（可能错过行情），系统在 `copy_engine.py` 内使用了 `ThreadPoolExecutor`。
- **降维打击**: 即使您追踪 10 个交易员，系统也会在不到 `1秒` 的并发时间内拿齐所有人的持仓数据。

## 2. 状态机与“持仓强制对齐”（自愈系统）

系统不是简单的“交易员开仓 -> 我就发请求开仓”。
网络总是会抖动，API 总会超时。为了解决这个问题，本系统引入了 **内存 + SQLite 数据库联动的排队状态机**。
- **快照追踪**: `copy_engine` 内存中保存了一份被验证过已经“发给交易所”的快照 (`_prev_snaps`)。
- **动态修正**: 系统每 `2.5 秒` 会对比最新抓到的交易员仓位（`curr_map`）和我们的内存快照（`prev_map`）。产生的增量就是 `new_list`（需要开的仓）和 `gone_list`（需要平的仓）。
- **断网自愈**: 如果 API 调用失败/超时，这笔单子**不会被踢出待办列表**，也不会写入 DB，它会在下个 2.5 秒的循环中继续被触发，直到成功 (`filled`) 或因为某种原因彻底跳过 (`skipped`)。
- **外挂补漏（强制对齐）**: 为防止严重错乱，系统每轮还会获取您的**账户真实持仓**（`get_my_positions`），如果发现您手里握着一个被追踪交易员“已平仓”的单子，会立即强行启动局部平仓补救。

## 3. 防重复开单机制（防重放/防双杀）

这是任何交易系统最致命的 BUG：下开仓单时 Bitget 服务器收到了，但在返回 200 OK 前您的网络断了。旧系统会在 2.5 秒后下第二单，导致重复开双倍仓甚至爆仓。
- **彻底防重 (`clientOid`)**: 系统为跟单交易生成了不可逆的唯一哈希追踪码：`hashlib.md5(f"{uid}_{tracking_no}").hexdigest()[:16]`。
- 每次 POST 到 `place-order` 接口，这串不可篡改的 ID 作为 `clientOid` 传给 Bitget 交易所。
- **即使网络断掉系统疯狂重试这笔开单，Bitget 交易所也会只认一笔，直接返回 HTTP 重复订单拦截**。这就是**绝对的幂等性保证**。

## 4. 多人冲突与精准“局部平仓”

系统支持追踪多个不同的交易员，当遇到极小概率事件：两位交易员同时开了**同一个币种，同一个方向**的仓位。
- **致命误区**: 绝对不能直接发一条指令说“把 BTCUSDT 的多单全平了”，这会立刻误杀另一个无辜交易员的持仓！
- **精确纪录 (`exec_qty`)**: 建仓成功时，系统会在 SQLite `copy_orders` 表中，以 `float` 形式记下我们为 A 交易员买下了 `0.045` 张大饼。
- **局部平仓**: 当 A 交易员平仓时，系统通过 `close_partial_position()`，精挑细选出只抛售这 `0.045` 张，绝不多平一丝一毫！
- **支持仓位模式**: 系统兼容 `单向持仓模式 (posMode=1)` 和 `双向持仓模式 (posMode=2)`。平仓时会自动下发不同的参数组合（如单向模式使用 `reduceOnly`，双向模式使用 `tradeSide=close`）。

## 5. 本金划拨与真实杠杆的计算规则

系统是如何把您的有限本金与交易员可能使用的100倍惊天杠杆结合的？

### 一单投入多少本金？
受严格物理隔离：`基础份额 = 跟单总本金 ÷ 追踪人数`
`本单下注额 (保证金) = 基础份额 × 设定的单笔最大比例 (max_margin_pct)`
**无论交易员买什么神仙币，或者开多少倍杠杆，您的本金亏损最大额度被绝对锁死在算出的这几十 USDT 之间。**

### 张数公式是怎么算的？
**系统永远复制该交易员该笔订单的“原味杠杆”。**
计算核心在 `order_executor._calc_size` 函数：
> `合约购买张数 = (您的单笔下注本金 × 交易员抓取到的真实杠杆) ÷ 资产当前市价`
> 计算出精准小数后截断为 4 位。

由于完全吸纳了交易员的杠杆倍数，您资金曲线的上下颠簸频率和爆仓线距，与该交易员本人的体验高度一致，但风险严格限制在您规定的本金敞口内。

## 6. 即时的无缝启停与人员增删

- **移除追踪人员**：网页一取消勾选，下一秒就会停止接收关于他的新号。对于还没平的旧有仓位，引擎的防御系统会在扫到您的真实账户时，立刻触发强制局部平仓机制，一并将他的余孽清理出局。
- **新增追踪人员**：网页一点上勾，下一秒他在 Bitget 公开页面上所有还持有着的现货/合约仓位，全部当做新增信号，瞬间帮您在账户里以市价建仓对标，“补票上车”。
