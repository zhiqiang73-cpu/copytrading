## 第二步：验证 WebSocket 连接（1分钟）

### 1. 查看启动日志

命令行窗口应该显示:
```
[WebSocket] 广播线程启动
吊 Web 仡盘：http://127.0.0.1:8080
```

✅ 看到这个说明 WebSocket 已启用！

### 2. 打开浏览器

访问: http://127.0.0.1:8080

### 3. 检查连接状态

**查看右上角心跳指示器:**

| 显示 | 含义 | WebSocket状态 |
|-----|------|--------------|
| 🟢 实时 | WebSocket 已连接 | ✅ 正常 |
| 🟡 重连中 | 正在重新连接 | ⚠️ 不稳定 |
| 🟡 在线 | 使用轮询模式 | ℹ️ 降级 |
| 🔴 离线 | 连接失败 | ❌ 异常 |

**成功标志**: 显示 **"🟢 实时"** 

### 4. 打开控制台（可选）

按 **F12** 打开开发者工具，切换到 **Console** 标签。

应该看到:
```
[WebSocket] 已连接
[WebSocket] 收到初始状态: {...}
```

✅ 看到这个说明 WebSocket 工作正常！

## 常见问题

### Q: 显示"在线"而不是"实时"

**可能原因**:
1. WebSocket 连接失败，自动降级到轮询
2. 浏览器不支持 WebSocket
3. 网络限制（代理/防火墙）

**解决方案**:
- 刷新页面重试
- 检查浏览器控制台是否有错误
- 确认使用 `python web.py` 启动（不是 `flask run`）

### Q: 控制台没有 [WebSocket] 日志

**原因**: Socket.IO 客户端未加载

**解决方案**:
- 检查网络连接
- 确认可以访问 CDN: https://cdn.socket.io
- 如果在内网环境，可下载 Socket.IO 离线版

### Q: 多窗口不同步

**原因**: 客户端未正确连接

**解决方案**:
- 查看后端日志是否有 "客户端连接" 记录
- 刷新所有窗口重新连接
