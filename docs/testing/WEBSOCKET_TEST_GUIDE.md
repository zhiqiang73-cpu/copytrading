# WebSocket 实时通信 - 快速测试

## 1. 安装依赖

```bash
pip install flask-socketio==5.3.6 python-socketio==5.11.0
```

## 2. 启动系统

```bash
cd "d:\MyAI\My work team\bitgetfollow"
python web.py
```

## 3. 查看启动日志

应该看到:
```
[WebSocket] 广播线程启动
```

## 4. 打开浏览器

访问: http://127.0.0.1:8080

### 检查连接状态

**右上角心跳指示器应该显示:**
- 🟢 "实时" (绿色边框) ← WebSocket 已连接

**打开浏览器控制台 (F12)，应该看到:**
```
[WebSocket] 已连接
[WebSocket] 收到初始状态: {engine: {...}, api_configured: true, ...}
```

## 5. 测试实时更新

### 测试 A: 引擎启动（单窗口）

1. 点击 "启动引擎" 按钮
2. **立即观察**（无需等待）:
   - ✅ 右上角出现 "引擎运行中" 绿色徽章
   - ✅ 页面弹出 "引擎已启动" 通知
   - ✅ 控制台显示: `[WebSocket] 引擎已启动`

**对比旧版本**: 需要等待 5 秒轮询才能看到更新

### 测试 B: 多窗口同步

1. 打开 **2个或更多** 浏览器标签页，都访问 http://127.0.0.1:8080
2. 在 **任意一个标签页** 点击 "启动引擎"
3. **所有标签页立即同步更新**:
   - ✅ 所有窗口都显示 "引擎运行中"
   - ✅ 所有窗口都收到通知

**这是 WebSocket 的威力！** 一个窗口操作，所有窗口实时同步。

### 测试 C: 网络中断恢复

1. 打开浏览器开发者工具 (F12)
2. 切换到 **Network** 标签
3. 选择 **Offline** (模拟断网)
4. 观察心跳指示器:
   - 🟡 "重连中" (黄色)
   - 控制台: `[WebSocket] 断开: transport close`
5. 恢复网络 (选择 **Online**)
6. 观察自动重连:
   - 🟢 "实时" (绿色)
   - 控制台: `[WebSocket] 已连接`

### 测试 D: 停止引擎

1. 点击 "停止引擎" 按钮
2. **立即观察**:
   - ✅ "引擎运行中" 徽章立即消失
   - ✅ 页面弹出 "引擎已停止" 通知

## 7. 常见问题

### Q: 心跳显示 "在线" 而不是 "实时"
**A**: WebSocket 未连接，降级到了轮询模式。
- 检查控制台是否有错误
- 确认使用 `python web.py` 启动（不是 `flask run`）
- 刷新页面重试

### Q: 控制台没有 `[WebSocket]` 日志
**A**: Socket.IO 客户端加载失败。
- 检查网络连接
- 确认 CDN 可访问
- 刷新页面

### Q: 多窗口不同步
**A**: 
- 检查控制台是否有连接错误
- 确认后端日志有 "客户端连接" 记录
