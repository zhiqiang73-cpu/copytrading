# 心跳机制和UI状态改进测试清单

## 改进内容

### 1. 心跳指示器 (顶部导航栏右侧)
- ✅ 添加了心形图标的心跳指示器
- ✅ 每3秒发送一次心跳 (原来是5秒)
- ✅ 显示三种状态:
  - 绿色 "在线" - 心跳成功
  - 黄色 "不稳定" - 1-2次心跳失败
  - 红色 "离线" - 连续3次心跳失败
  - 红色 "超时" - 超过15秒无响应
- ✅ 心跳成功时有脉搏动画效果

### 2. 引擎运行状态显示 (顶部导航栏)
- ✅ 引擎运行时显示绿色 "引擎运行中" 徽章
- ✅ 引擎停止时自动隐藏
- ✅ 每5秒自动更新状态

### 3. 提示消息改进
- ✅ 成功消息保持8秒 (原来3秒)
- ✅ 错误消息保持15秒 (原来3秒)
- ✅ 信息消息保持5秒 (原来3.5秒)
- ✅ 添加手动关闭按钮(×)
- ✅ 消息框支持自定义内容(不仅仅是纯文本)

### 4. 引擎控制按钮改进
- ✅ 点击启动/停止时禁用按钮,防止重复点击
- ✅ 操作完成后自动刷新持仓和订单数据
- ✅ 立即更新状态徽章
- ✅ 显示更详细的成功/失败消息

### 5. 状态同步
- ✅ 页面加载时检查实时引擎状态
- ✅ 每5秒自动检查引擎状态
- ✅ 顶部导航栏状态与页面内状态同步

## 测试步骤

### 测试1: 心跳指示器
1. 打开浏览器访问 http://127.0.0.1:8080
2. 观察右上角的心形图标指示器
3. 应该显示绿色"在线"状态
4. 每3秒左右应该有一次脉搏动画(心形图标闪烁)
5. 关闭后端服务,等待几秒后应该变成黄色"不稳定"
6. 再等待几秒应该变成红色"离线"

### 测试2: 引擎状态显示
1. 在跟单管理页面点击"启动"按钮
2. 观察顶部导航栏应该出现绿色"引擎运行中"徽章
3. 点击"停止"按钮
4. 徽章应该自动消失
5. 刷新页面,如果引擎在运行,徽章应该自动显示

### 测试3: 提示消息持久性
1. 点击"启动"按钮
2. 观察页面顶部的成功提示消息
3. 消息应该保持约8秒(而不是快速消失)
4. 可以点击消息上的×按钮手动关闭
5. 测试一个错误操作(如未配置API就启动)
6. 错误消息应该保持约15秒

### 测试4: 按钮防重复点击
1. 快速连续点击"启动"按钮多次
2. 按钮应该在第一次点击后变灰,无法再次点击
3. 操作完成后按钮应该恢复可点击状态

### 测试5: 状态实时更新
1. 打开两个浏览器标签页,都访问跟单管理页面
2. 在第一个标签页点击"启动"
3. 等待5-10秒
4. 观察第二个标签页的状态徽章应该自动更新为"运行中"

## 预期行为

✅ **心跳正常**: 绿色心形图标,每3秒闪烁一次,显示"在线"
✅ **引擎运行**: 顶部导航栏显示绿色"引擎运行中"徽章
✅ **提示持久**: 成功消息显示8秒,错误消息显示15秒,可手动关闭
✅ **防重复点击**: 按钮点击后禁用,操作完成后恢复
✅ **状态同步**: 多个标签页的状态会自动同步

## 问题排查

如果心跳显示"离线":
- 检查后端是否正常运行
- 打开浏览器控制台(F12)查看是否有错误
- 确认没有CORS或网络问题

如果引擎状态不更新:
- 刷新页面
- 检查浏览器控制台是否有JavaScript错误
- 确认API `/api/status` 可以正常访问

如果消息快速消失:
- 检查浏览器控制台是否有JavaScript错误
- 确认代码更新已生效(清除浏览器缓存后刷新)
