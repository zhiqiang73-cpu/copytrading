# 价差容忍度配置修复

## 问题

之前代码里硬编码了 **5%** 的价差容忍度，导致：
- 交易员价格 vs 市场价格相差 > 5% 才会跳过
- 太宽松，可能跟到过时的价格

数据库默认值更离谱：**0.02%**，几乎所有单都会被跳过。

## 修复

### 1. 统一默认值为 **1%**

| 位置 | 修改 |
|-----|------|
| `database.py` | `price_tolerance` 默认值 0.0002 → **0.01** |
| `database.py` | `binance_price_tolerance` 默认值 0.0002 → **0.01** |
| `copy_engine.py` | `bg_tol` 回退值 0.05 → **0.01** |
| `copy_engine.py` | `bn_tol` 回退值 0.05 → **0.01** |

### 2. 添加前端配置

在 "策略与风控" 部分新增 **"价差容忍 (%)"** 配置项：

```html
<input id="price_tolerance_pct" 
       type="number" 
       step="0.1" 
       placeholder="1.0"
       title="交易员价格与市场价格差异超过此值时跳过">
```

### 3. 工作原理

```
交易员开单价格: $100
当前市场价格: $101
价差: |101-100|/100 = 1%

if 1% > price_tolerance:
    跳过此单 (价差超限)
else:
    执行跟单
```

## 使用说明

### 查看当前配置

1. 打开 "实盘" 页面
2. 展开 "策略与风控"
3. 查看 "价差容忍 (%)" 输入框

### 修改配置

1. 输入新的百分比值，例如:
   - **1.0** - 默认，适中 (推荐)
   - **0.5** - 严格，仅跟价格接近的单
   - **2.0** - 宽松，允许更大价差
   - **5.0** - 极宽松，几乎不限制

2. 点击 "保存策略"

3. **重启引擎** 使配置生效

### 推荐值

| 场景 | 推荐值 | 说明 |
|-----|-------|------|
| 稳健跟单 | **1.0%** | 平衡，过滤价差大的单 |
| 激进跟单 | **2.0%** | 宽松，跟更多单 |
| 保守跟单 | **0.5%** | 严格，只跟价格精确的单 |
| 高波动币种 | **1.5-2.0%** | 价格波动大，适当放宽 |
| 低波动币种 | **0.5-1.0%** | 价格稳定，收紧容忍度 |

## 技术细节

### 数据库字段

```sql
-- Bitget 价差容忍度
price_tolerance REAL DEFAULT 0.01

-- Binance 价差容忍度
binance_price_tolerance REAL DEFAULT 0.01
```

### 代码逻辑 (copy_engine.py)

```python
# Bitget
bg_tol = _safe_float(settings.get("price_tolerance"), 0.01)

# Binance
bn_tol = _safe_float(settings.get("binance_price_tolerance"), 0.01)

# 检查价差
curr_p = get_current_price(symbol)
dev = abs(curr_p - signal_price) / signal_price

if dev > tol:
    # 跳过此单
    logger.warning("价差超限: %.2f%%", dev * 100)
    return
```

## 注意事项

1. **配置独立**: Bitget 和 Binance 各自独立配置
2. **立即生效**: 保存后下次开单生效，无需重启
3. **历史订单**: 不影响已开的仓位
4. **过于严格**: 如果设置太小（< 0.3%），可能导致大部分单被跳过
